# Backend

## Runtime Architecture

Main entrypoint: `src/index.ts`.

At activation, the plugin:

1. Loads and merges config (OpenClaw config + env overrides)
2. Validates config schema and hook bridge constraints, then aborts activation on invalid config
3. Initializes Hook Bridge runtime
4. Initializes transport ownership manager
5. Starts owner-only services (WS server, queue, event log) when role is owner
6. Registers internal hooks and typed plugin hooks

## Code Structure

- `src/index.ts`: plugin lifecycle, activation/deactivation, hook registration.
- `src/runtime/`: runtime orchestration.
  - `internal-handlers.ts`: internal hook mapping (`message:*`, `command:*`, etc.).
  - `register-*.ts`: typed hook mapping (`before_tool_call`, `session_start`, etc.).
  - `runtime-events.ts`: transport pipeline, filtering, redaction/signing, queue/webhook dispatch.
  - `hook-bridge*.ts`: automation bridge and tool guard engine.
- `src/hooks/`: canonical event builder functions + trackers.
- `src/transport/`: single-owner lock/relay protocol.
- `src/broadcast/`: WS server, HTTP webhook sender, queue.
- `src/config/`: defaults, schema, env parsing, validation.
- `src/events/`: envelope types + redaction/signing.
- `src/logging/`: runtime logger and NDJSON event/runtime file logger.

## Processing Pipeline

The typical pipeline for an event:

1. Hook event enters from OpenClaw (`registerHook` or `on`).
2. Hook handler converts raw payload into canonical event builder input.
3. Canonical event is created (`createCanonicalEvent`), with IDs/correlation.
4. Runtime transport dispatch decides owner/follower handling.
5. Owner path applies:
   - Hook Bridge async rule checks (non-toolguard)
   - Redaction
   - HMAC signing (optional)
   - Event file logging
   - WebSocket broadcast
   - Webhook filtering + queue/direct webhook delivery

## State Management

`PluginState` tracks:

- `toolTracker`: in-flight tool calls for duration/correlation
- `pendingToolCalls`/`pendingToolCallsByContext`: map before/after tool hooks
- `sessionTracker`: session/agent identity mapping
- `statusReducer`: synthetic `agent.status` transitions
- `subagentTracker`: subagent spawn/idle transitions
- `transportManager`: current ownership role and relay
- `hookBridge`: rules engine for async actions + tool guard

## Synthetic Events

Synthetic events are generated by plugin logic, not raw OpenClaw hooks:

- `agent.activity`
- `agent.status`
- `agent.sub_agent_spawn`
- `subagent.idle`

## Reliability Model

- In `auto/owner/follower` modes, one runtime owns transport and others relay.
- Owner deduplicates relayed events by `eventId` and semantic key.
- Queue is optional but used for webhook durability and retry control.
